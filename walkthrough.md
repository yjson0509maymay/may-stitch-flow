# WriteFlow 랜딩 페이지 구현 요약 (Walkthrough)

## 1. 개요 (Overview)
이 문서는 사용자가 제공한 데스크톱 랜딩 페이지 디자인(`screen.png`)을 바탕으로 `stitch_writeflow/my-app`에 Next.js와 Tailwind CSS v4를 활용하여 구현한 작업 내역을 설명합니다.

## 2. 디자인 시스템 및 환경 설정
가장 먼저 앱 전역에 일관된 스타일을 적용하기 위해 `app/globals.css`를 수정했습니다.
* **색상 팔레트:** 메인 브랜드 컬러인 오렌지색(`#F26115`)과 텍스트 기본색(`#1E293B`), 각종 배경색(`gray`, `light-blue` 등)을 커스텀 CSS 변수로 추가했습니다.
* **배경 효과:** 
  * `.gradient-hero`: 메인 히어로 섹션에 쓰이는 위에서 아래로 내려오는 부드러운 파란빛 그라데이션.
  * `.gradient-cta`: 하단 CTA 섹션에 쓰이는 오렌지에서 초록색으로 이어지는 좌우 그라데이션 클래스를 선언했습니다.
* **아이콘 및 이미지:** 
  * `lucide-react`를 설치하여 화면 전반에 사용된 아이콘(Check, Users, PlayCircle 등)을 손쉽게 구성했습니다.
  * 히어로 섹션 내의 노트북 목업 이미지는 AI 이미지 생성 기능을 사용해 제작한 뒤 `public/hero.png`로 배치했습니다.

## 3. UI 컴포넌트 구조 (`app/page.tsx`)
화면은 크게 **헤더, 히어로 섹션, 기능 소개, 요금제, CTA, 꼬리말(Footer)** 6가지 영역으로 나뉘어 구현되었습니다.

### 3.1 헤더 (Header)
* 로고 텍스트, 주요 네비게이션 링크, 로그인/무료 체험 시작 버튼을 좌우로 넓게 배치하여 전형적인 랜딩 페이지 헤더를 구현했습니다.

### 3.2 메인 히어로 섹션 (Hero Section)
* 시선을 끄는 대형 텍스트와 강조 색상을 적용했습니다.
* 둥근 창 스타일(Mac OS 스타일의 빨강, 노랑, 초록 버튼)을 CSS로 구현하고, 내부에 `next/image`를 사용해 노트북 목업 이미지를 꽉 채워(aspect-video) 배치했습니다.

### 3.3 기능 소개 (Features Section)
* CSS Grid(`grid-cols-3`)를 사용하여 6개의 기능 카드를 2줄로 배치했습니다.
* 각 아이콘 컨테이너는 연한 파란색 배경을 주어 아이콘 색상(주황, 녹색, 파랑 등)이 돋보이도록 디자인했습니다.
* 텍스트에 `break-keep`을 적용해 한국어 텍스트의 가독성을 높였습니다.

### 3.4 요금제 (Pricing Section)
* 3개의 가격 플랜(Free, Pro, Enterprise)을 카드 형태로 구성했습니다.
* **강조 효과:** 'Pro' 플랜 카드의 경우 테두리 색상을 오렌지색으로 굵게 주고, 상단에 '가장 인기' 배지를 띄운 뒤 `transform: translateY`를 적용해 시각적으로 튀어나와 보이도록 구현했습니다.

### 3.5 하단 전환 (CTA) 및 푸터 (Footer)
* CTA 영역은 텍스트 컬러를 흰색으로 반전시키고, 배경으로 화려한 그라데이션 클래스(`.gradient-cta`)를 적용하여 강한 인상을 주도록 만들었습니다.
* 푸터는 로고와 카피라이트, 간단한 정책/도움말 링크로 깔끔하게 마무리했습니다.
